Hello Git

1.0 Git

​ 在官网下载安装，，不需要管安装版本

​ 安装完成，无桌面图标，任意项目中右键，出现 git GUI here 和 git bash here 代表安装成功

​

​ 在使用 git 之前，需要指定操作的作者信息

git config --global user.name 'Kris Wu'
git config --global user.email 'xiaoming@163.com'

​ 注意：邮箱只是需要格式，不是一定要正确的邮箱，后面的引号可以加可以不加，如果有空格一定要加

​ 使用 git 的操作是要干什么

​ git 是进行版本控制的，也就是说，将来我们的项目会每天修改，每天增加新功能，那么导致一些问题，就是如果不记录版本或者没有人帮我们存储进度，那么要返回到之前的操作或者和其他同事协同开发就会很复杂。

​

​ 通过 git 工具，可以帮我们在本地项目中生成一个仓库，这个仓库可以记录我们每次修改后的提交信息，记录我们每一次的操作，那么如果想回到哪一次的操作，即可根据日志的 id 回到那一次，而不是像以前那样撤销，因为有可能是昨天的操作，无法撤销了。

2.0 第一次创建项目

​ 一定要在当前项目中右键，打开 git bash here

git init
git status
git add ./
git commit -m '第一次提交'
git log

​ 从暂存区恢复文件覆盖工作区的文件

git checkout '指定的文件名称或者路径'

​

​ 工作区是指的编辑器开发的地方，

​ 暂存区是临时存储的地方，可以有一些小操作

​ 存储区，是每一次 add 和 commit 进行存档的地方，每次存档的内容由仓库分配

git rm --cached 指定的文件

​ 把暂存区的文件从该区撤销到工作区，但是如果撤销下来，文件是新建的，就会显示为红色

​ 所以一般如果不需要添加到仓库的文件，需要设置忽略名单

git log --oneline             记录简洁的版本号和提交信息
git reset --hard 指定的版本号   版本回退
git reflog                    所有的提交信息

​ 可以在任意版本穿梭

3.0 分支

​ 假设在一条生产线上，需要生产一台手机

​ 如果由单个人来完成，时间成本会比较高，一个人处理的业务量也会比较大

​ 创建分支，也就是创建新的生产线，这条生产线的内容和上一个一样

​ 那么上个人就可以交付一部分的功能由此人完成

​ 在最终，二者进行合并功能，即可完成此次的开发，节省了效率

git branch              查看当前的分支
git branch 分支名      创建新分支
git checkout 分支名    切换分支
git merge 分支名

细节： 在每一次切换分支的时候，一定要对当前的代码进行存档 add commit

注意： 如果进入了奇怪的模式，试试英文模式下 :q 或者 ctrl +c 再或者 esc

只有 master 主分支上才可以合并分支！！！

只有在 master 分支上才可以删除，只有在提交之后才可以删除，不然就强制删除

情景：如果在当前分支开发，领导临时通知测试功能，需要暂停开发

​ 有可能之前写的东西有 bug，有可能要给别人演示功能，如果我们分支操作不存档，在主分支就会提示有未完成的功能无法合并或者提交到远程仓库等等…

​ 在当前的分支上暂停开发，保存代码

git add ./      添加到暂存区
git stash       保存当前的状态

​ 去参与到测试或者其他功能创建

​ 如果完成了所有操作，恢复之前的开发进度

git stash pop    恢复开发进度

​ 有人说，直接 add 和 commit 就行了，就没这么多事儿了，确实

​ 不过也有可能，你提交的东西还是未完成的，执行的话可能会报错…

4.0 github 的操作

​ 注册 - 激活 - 创建仓库，千万不要点击创建 readME 文件

​ 线下创建的仓库，和线上默认的仓库如果同时存在，会导致我们的项目上传不上去

​ 而默认创建 README 文件，就会在远程仓库创建一个默认的仓库

​ 我们的项目默认都是公开的，所有人都可以克隆或者下载，但是不是所有人都可以修改

​ 在开发中，一般为了多人协同开发，但是账号密码也不会共享，所以会有多个账号合作

​ 在主开发人员的项目中，点击 settings 找到 coll 开头的单词，添加另外一个人的账号

​ 将邀请链接发给另外一个人，这样，他同意邀请就可以参与到项目的提交了

​ 如果另外的这个人要提交，也是要输入自己的账号和密码的

凡凡创建了仓库，搭建好了项目的基本环境，上传到了远程仓库

git init                              初始化仓库
git add ./                            添加到暂存区
git commit -m '初始化项目'                             添加到存储区

git remote add origin 项目地址         设置变量指向远端映射的地址
git push -u origin master             记住当前的变量，将来简洁推送

​ 如果是第一次提交项目，需要在弹出的面板中输入账号和密码

​ 如果输入错误，也会记录，导致上传不成功。需要在控制面板中找到 windows 凭证管理器删除记录

​ 有时候因为网络原因也会上传不成功，切换网络试试

​ 如果担心是密码的原因，可以在网页上退出登录，在重新登录试试

坤坤是新来的，需要获取线上的项目

git clone 项目地址

git branch kunkun
git checkout kunkun

// 或者一行一步到位
// git checkout -b kunkun

// 如果完成了新功能
git add ./
git commit -m '坤坤完成了新功能'

// 或者一行一步到位
git commit --all -m '如果是新建的文件就不行，只有是基于源文件修改才可以'

git remote add myset 远程地址
git push -u myset kunkun

// 回到主分支合并
git checkout master
git merge kunkun

git push

​

配置个人主页

​ 新建仓库，仓库的名称是 账号.github.io

​ 将本地的代码上传到远程仓库，然后访问地址 账号.github.io 有时候网不好，多刷新刷新

​ 如果访问不了，一定要把 http 换成 https，有些浏览器默认用 http 访问

​ 这里的个人主页只能上传静态的资源文件，不能上传全栈项目，后端文件在这里运行不起来

​ 如果你有线上的服务器存储和运行了后端文件，或者你项目中是爬取的数据，就可以运行

演示码云上传项目

​ 过程和 GitHub 上传项目是一样的

​ 在 GitHub 中删除项目需要找到 settings 的最后一项

​ 在码云中删除项目，找到 delete 专栏

​ 分支的项目上传完毕之后，可以在上传项目的分支结构中查看
